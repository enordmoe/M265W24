Logical Vectors: The Essentials

  1. Logical vectors are those that can take on only TRUE, FALSE and NA values.

    • They can indicate presence or absence of a trait (some binary variables are logicals).

    • They result from logical operations

  2. Logical operations include comparisons (e.g., <, ==, etc.) and Boolean algebra operations such as & and |.

  3. Logical vectors underlie our use of the filter() function.

  4. Logical vectors are also used in conditional operations, such as creating new variables via mutate() based on certain conditions.

    • Logical operations also allow for combining logical conditions to enable more complex filtering and creation of variables based on multiple characteristics.

  5. Data summaries can be produced relying on careful and sometimes creative use of statistical operations combined with logical variables, e.g. summarize(number_seniors = sum(age >= 65)

Wisdom on NAs

The unknown unknowns

“Reports that say that something hasn’t happened are always interesting to me, because as we know, there are **known knowns*8; there are things we know we know. We also know there are known unknowns; that is to say we know there are some things we do not know. But there are also unknown unknowns-the ones we don’t know we don’t know. And if one looks throughout the history of our country and other free countries, it is the latter category that tends to be the difficult ones.”

Donald Rumsfeld, former Secretary of Defense (and Data Scientist?)

R4DS example illustrates why

# We don't know how tall Olds was
height_olds <- NA
height_olds
[1] NA
# We don't know how tall Upton was
height_upton <- NA
height_upton
[1] NA
# Were Olds and Upton the same height?
height_olds == height_upton
[1] NA
  • We don’t know (unless we have a picture of them together?)
  • We can’t say NA == NA because NA typically indicates uncertainty.

Boolean and Missing Values

Combining logicals with missing (NA) values can be treacherous as this example from R4DS illustrates:

df <- tibble(x = c(TRUE, FALSE, NA))

df |> 
  mutate(
    and = x & NA,
    or = x | NA
  )
# A tibble: 3 × 3
  x     and   or   
  <lgl> <lgl> <lgl>
1 TRUE  NA    TRUE 
2 FALSE FALSE NA   
3 NA    NA    NA
  • For an & to be TRUE, both conditions have to be knowably TRUE
  • For an | to be TRUE, at least one condition has to be knowably TRUE

Conditional transformation: case_when()

Use case_when() for more complicated transformations (beware of NAs)

  • Create a new timeofday variable
  • Value assigned based on first TRUE condition
chiflights2020 |> 
  mutate(
    timeofday = case_when(
      is.na(dep_time)      ~ "cancelled",
      dep_time < 0300       ~ "late night",
      dep_time < 0700       ~ "early morning",
      dep_time < 1100       ~ "morning",
      dep_time < 1400       ~ "midday",

      dep_time < 2200       ~ "evening",

Summary

  • Logical vectors are fundamental in data science, consisting of TRUE, FALSE, and NA.
  • Logical operations include comparisons (>, ==) and Boolean algebra (&, |).
  • Used extensively in filtering (filter()), summarizing (summarize()), and conditional transformations (mutate() with ifelse() or case_when()).
  • Logical vectors interact with missing values (NA), which can complicate comparisons and Boolean operations.
  • Chicago Flights Dataset demonstrates logical filtering, summarization, and transformation in real-world data.
  • Summarizing: summarize() can use logical conditions to compute counts (sum(condition)), proportions (mean(condition)), and group-based logic (any(), all()).
